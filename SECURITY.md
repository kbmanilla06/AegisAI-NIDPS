# Security Policy

## Scope and responsible use

AegisAI NIDPS is defensive. Use only synthetic, public, locally owned, or explicitly authorized data and environments. Sprints 0–9 support simulated prevention only.

## Reporting a vulnerability

Until a private reporting channel is published, do not post sensitive exploit details or real telemetry in public issues. Contact the repository owner privately and include affected version, impact, safe reproduction, and suggested mitigation without credentials or unauthorized data.

## Prohibited content and behavior

No credentials, tokens, private PCAPs, malware, offensive payloads, unauthorized targets, secret environment files, real enforcement configuration, or destructive test instructions belong in the repository.

## Security gates

Critical/High issues, broken authorization, unsafe parsing/artifacts, secret exposure, missing audit, or any real/automatic prevention capability block release. See the threat model, test strategy, and prevention safety model.
